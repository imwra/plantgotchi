# Plantgotchi iOS + Android Mobile Apps Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build native iOS (Swift/SwiftUI) and Android (Kotlin/Jetpack Compose) apps that connect to Plantgotchi BLE sensors, display plant health dashboards, run offline rule engines, and sync data to Turso cloud.

**Architecture:** Both apps follow the same pattern: CoreBluetooth/BLESSED for BLE sensor discovery, local GRDB/Room databases mirroring the web schema, an offline rule engine ported from TypeScript, and Turso HTTP sync for cloud connectivity. The phone acts as the smart node вЂ” sensors are dumb BLE peripherals.

**Tech Stack:**
- **iOS:** Swift 5.10+, SwiftUI, CoreBluetooth, GRDB.swift v7, URLSession (Turso HTTP API)
- **Android:** Kotlin 2.0+, Jetpack Compose, BLESSED-android, Room, Ktor/OkHttp (Turso HTTP API)
- **Shared:** Same SQLite schema, same BLE GATT UUIDs, same rule engine logic

**Constraints:** No Xcode or Android SDK on build machine. All code is written but cannot be compiled. User finishes build on Mac (iOS) / any machine with Android Studio (Android).

## What the user needs to finish

### iOS (on Mac with Xcode)
1. Open `ios-app/` directory in Xcode
2. Xcode auto-resolves SPM dependencies (GRDB)
3. Add "Press Start 2P" font to asset catalog
4. Build + Run on simulator/device
5. Fix any compiler issues (expected to be minimal вЂ” all logic is pure Swift)

### Android (any machine with Android Studio)
1. Open `android-app/` in Android Studio
2. Gradle syncs dependencies automatically
3. Download "Press Start 2P" TTF into `res/font/`
4. Build + Run on emulator/device
5. Run `./gradlew test` for unit tests
